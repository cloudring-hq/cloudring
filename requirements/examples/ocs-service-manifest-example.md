# Synthetic OCS Service Manifest Example

This is a filled product-level example for
[../templates/ocs-service-manifest-template.md](../templates/ocs-service-manifest-template.md).
It is not the final schema format.

```yaml
manifest:
  standard_version: ocs-0.1-example
  manifest_maturity: draft
  service:
    id: service-portable-a
    name: Portable Synthetic Service
    purpose: provide a small stateful business capability through CloudRING
    owner: org-owner-a
    publisher: publisher-isv-a
    support_owner: support-owner-a
    tenant_scope: dedicated
    lifecycle_state: candidate
  users:
    - user-buyer-a
    - admin-private-a
    - agent-builder-a
  product_promises:
    - can be created and validated locally
    - can be installed in private store when private-ready evidence passes
    - exposes honest portability limits before order or install
  non_goals:
    - does not claim global-ready status
    - does not claim automatic migration for every dependency
  requirements:
    requirement_refs:
      - CR-OCSCONTRACT-001..046
      - CR-SPECTPL-003..004
      - CR-SPECEX-003
    adr_refs:
      - ADR-0002
      - ADR-0008
    conformance_refs:
      - stage1-service-ready
      - stage3-private-store-ready
  capability:
    required:
      - standard lifecycle
      - health and readiness
      - supportable observability
      - portability statement
    optional:
      - embedded UI
    unsupported:
      - global settlement until later stage
    extensions:
      - namespace: synthetic.extension.example
        owner: org-owner-a
        compatibility_impact: optional only
  profiles:
    - profile_id: local
      status: supported
      overrides:
        public_values:
          - service display name
        secret_references:
          - synthetic-dev-secret-ref owned by org-owner-a
        generated_values:
          - local route generated by local profile
      policy:
        data_classes:
          - synthetic application data
        allowed_locations:
          - local synthetic presence
        forbidden_locations:
          - undeclared target
        required_approvals:
          - none for local validation
    - profile_id: private
      status: degraded
      overrides:
        public_values:
          - private support tier
        secret_references:
          - synthetic-private-secret-ref owned by admin-private-a
        generated_values:
          - private route generated by private profile
      policy:
        data_classes:
          - synthetic application data
        allowed_locations:
          - presence-private-a
        forbidden_locations:
          - target without policy profile
        required_approvals:
          - admin approval for stateful restore test
  components:
    - id: api
      role: api
      owner: publisher-isv-a
      lifecycle: managed
      readiness:
        health: declared
        observability: declared
    - id: data
      role: database
      owner: admin-private-a
      lifecycle: managed
      readiness:
        health: declared
        observability: limited
  dependencies:
    - id: synthetic-object-store
      capability: object storage
      connection_contract:
        outputs:
          - endpoint reference
          - bucket identity
        secret_boundary: brokered
        degraded_state: upload disabled, read-only data remains available
        fallback: manual
        owner: admin-private-a
  lifecycle_actions:
    - action: validate
      risk_class: read-only
      idempotency: required
      approval: none
      result_shape: standard lifecycle result
    - action: provision
      risk_class: controlled-change
      idempotency: required
      approval: owner
      result_shape: standard lifecycle result
    - action: export
      risk_class: controlled-change
      idempotency: explicit-repeat-consequence
      approval: policy
      result_shape: standard lifecycle result
  usage:
    resources:
      - id: usage-resource-a
        unit: synthetic request
        scope: service
        billing_relevance: informational
        exclusions:
          - health checks
  health_observability:
    health: synthetic health contract
    readiness: synthetic readiness contract
    metrics:
      - request count
      - error count
    logs:
      - structured operational log, redacted
    traces:
      - request correlation
    events:
      - lifecycle state change
  policy_and_trust:
    placement_constraints:
      - must satisfy policy-profile-a
    data_residency:
      - synthetic data remains in approved presence
    audit_requirements:
      - lifecycle action audit
      - export action audit
    certification_state: private-ready
    trust_state: trusted
  portability:
    status: assisted
    exportable_data:
      - synthetic application data
    exportable_metadata:
      - service configuration metadata without secrets
    restore_or_import_story: evidence-bundle-a
    compatible_targets:
      - migration-target-a
    known_limits:
      - stateful dependency needs restore validation before migration
  ui:
    embed_mode: both
    descriptor_ref: synthetic-ui-descriptor
    scoped_context:
      - tenant identity reference
      - service instance reference
    allowed_actions:
      - view status
      - request export plan
    support_owner: support-owner-a
  docs:
    overview: synthetic-doc-overview
    api_contract: synthetic-api-contract
    architecture: synthetic-architecture-note
    runbook: synthetic-runbook
    faq: synthetic-faq
    known_limits: synthetic-known-limits
  generated_artifacts:
    - id: local-runtime-config
      source_of_truth: manifest profile and field matrix
      generator: synthetic generator
      target_profile: local
      freshness: current
      publish_boundary: local-only
      cleanup_rule: required
  evidence:
    conformance_report: synthetic-stage1-report
    validation_summary: synthetic-validation-summary
    source_safety: passed
    evidence_owner: org-owner-a
    review_trigger: manifest version change
  source_safety:
    sensitivity_class: public-template
    redaction_status: safe
    copy_risk_status: paraphrased
    forbidden_content_result: passed
    reviewer_or_owner: org-owner-a
    stop_if_unknown: true
```

## Non-Claims

- This example does not choose final schema syntax.
- This example does not prove runtime execution.
- This example does not contain real customer, provider, source or operational
  data.
