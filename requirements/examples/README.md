## Rules

1. Examples show product meaning, not final implementation format.
2. Examples must use only synthetic objects from
   [../scenarios/synthetic-fixture-catalog.md](../scenarios/synthetic-fixture-catalog.md)
   or generic safe IDs.
3. Examples must include source-safety fields.
4. Examples must include non-claims where evidence is partial.
5. Examples must not contain real source paths, provider names, customer names,
   endpoints, network addresses, credentials, tenant data or source snippets.
