# Directory Media Batch Context

This run indexes media and design artifacts without copying raw paths, file names, EXIF text, OCR text, URLs, domains, provider names, people, brand names, or secrets. Raster images receive safe metadata buckets; vector files are inspected only for controlled structure signals; audio, video, and design-source files are retained as explicit future-processing backlog.

## Agent Use

- Use `media-cases.jsonl` for media/design evidence about UI, diagrams, demos, training, support proof, and product visuals.
- Use `media-signals.jsonl` for stable-id traceability and safe metadata buckets only.
- Treat future-processing statuses as OCR/transcription/visual-review backlog, not as missing coverage.
- Do not infer brand identity from visual assets unless a future redacted visual pipeline explicitly provides safe tags.
